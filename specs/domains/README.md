# Доменная Архитектура (Domain Driven Design)

Эта директория содержит спецификации бизнес-логики, разделенные по предметным областям (Bounded Contexts). Каждый подкаталог представляет собой независимый домен со своими сущностями, правилами и процессами.

## Структура Домена

Каждый домен обычно содержит следующие спецификации:

- **models.md**: Определение сущностей, агрегатов и объектов-значений (Value Objects). Здесь описывается структура данных и инварианты.
- **flows.md**: Описание бизнес-процессов, сценариев использования и диаграмм последовательности. Здесь описывается поведение системы.

|-------|----------|---------------|---------------------|

## Принципы

1. **Изоляция**: Домены должны быть максимально независимы друг от друга.
2. **Контракты**: Взаимодействие между доменами происходит через четко определенные интерфейсы или события.
3. **Единый язык**: Используйте терминологию, понятную как разработчикам, так и бизнес-заказчикам (Ubiquitous Language).

# Протокол Mentor (Mentor Protocol)

Это корневая директория, содержащая все Protobuf определения для проекта Mentor. Библиотека определяет контракты взаимодействия между микросервисами, клиентскими приложениями и административной панелью.

## Структура Пакетов

Библиотека разделена на логические уровни для обеспечения чистоты архитектуры:

### 1. [Common Types (Общие Типы)](./types/)
**Директория**: `mentor/types/`
Содержит переиспользуемые модели данных (Messages, Enums), которые являются общими для всех сервисов. Здесь определяются сущности предметной области (User, Bot, Chat, Transaction).

### 2. [Client API (Клиентский API)](./client/)
**Директория**: `mentor/client/`
Публичный API для конечных пользователей (Mobile App, Web App, Mini App).
*   Авторизация и профиль (`identity`).
*   Каталог ботов и библиотека (`market`).
*   Обмен сообщениями (`chat`).

### 3. [Admin API (Административный API)](./admin/)
**Директория**: `mentor/admin/`
Закрытый API для Backoffice, CMS и администраторов платформы.
*   Управление пользователями.
*   Настройка и модерация ботов.
*   Управление биллингом и тарифами.

## Стандарты Разработки

*   **Язык**: Все комментарии и документация строго на **Русском языке**.
*   **Стиль**: Google Protobuf Style Guide.
*   **Индексация полей**:
    *   Поля моделей **не удаляются** (только помечаются `deprecated`).
    *   Номера полей (`tag numbers`) **не переиспользуются**.
*   **Версионирование**: Соблюдается обратная совместимость.

## Сборка и Генерация

Для генерации кода (Go, TypeScript, Rust) используется `buf` или `protoc` с соответствующими плагинами. См. корневой `README.md` для инструкций по запуску.
